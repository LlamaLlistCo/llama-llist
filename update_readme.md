# Llama Llist 智能化与羊驼主题升级说明

## 一、升级目标

Llama Llist 从普通“灵感笔记/待办清单”升级为更有品牌识别度的羊驼主题效率工具。应用继续保留笔记、标签、待办、搜索、模板、图片、AI 整理等核心业务，同时强化：

- 羊驼主题装扮：让软件名称、视觉风格和页面资产统一。
- AI 业务闭环：端侧提取、云端整理、结果可编辑并写回笔记/待办。
- 异常降级：断网、云端超时、空返回时仍保留端侧整理结果。
- 数据稳定性：索引、关联清理、模板字段和默认模板补全。
- 待办体验：点击圆圈即时完成/恢复，四象限优先级圆点独立调整。

## 二、主要改动

### 1. 羊驼主题与页面视觉

新增 6 套主题：羊驼草甸、安第斯纸本、云朵羊毛、山谷晨光、极简牧场、AI 驼队工作台。每套主题包含主色、卡片色、柔和底色、羊驼 mascot 和主题口号。

新增图片资源：

- `llama_mascot.png`：羊驼形象，用于首页、任务页和管理页。
- `llama_meadow_bg.png`、`llama_wool_bg.png`、`llama_andes_bg.png`：主题背景。
- `note_bg_meadow.png`、`note_bg_andes.png`、`note_bg_wool.png`、`note_bg_sunrise.png`、`note_bg_ranch.png`、`note_bg_caravan.png`：互不重复的笔记背景。

首页已重构为更接近参考图的仪表盘布局：顶部插画 Hero、今日概览三卡、主题化标签筛选、带封面的灵感卡片。笔记编辑页新增“笔记背景”选择器，使用 `view_type` 保存背景类型；若笔记插入了图片，首页卡片优先展示插入图片，否则展示所选背景图。

### 2. 待办页面与状态逻辑

待办页重构为 `Llama Peak` 任务工作台：

- 顶部新增“今日任务牧场”概览卡，展示待处理、已延期、紧急任务数量。
- 状态筛选和优先级筛选拆分成清晰的横向筛选栏。
- 任务卡片改为紧凑列表：左侧圆圈负责完成/恢复，右侧四个彩色圆点负责紧急程度切换。
- 空状态改为羊驼插图，不再依赖简单 emoji。

状态规则统一为：

- 点击未完成任务前方圆圈：立即变为 `completed`。
- 点击已完成任务前方圆圈：根据截止日期恢复为 `pending` 或 `delayed`。
- 点击四个彩色圆点：只修改 `priority`，不修改 `status`。
- UI 使用乐观更新：点击后先本地更新列表，接口失败再回滚并提示。
- 列表渲染 key 包含 `id + status + priority`，避免状态改变后卡片不刷新。

### 3. 模板功能增强

后端 `templates` 表新增字段：

- `description`：模板说明。
- `sort_order`：排序。
- `is_system`：是否系统模板。

默认模板扩展为草甸灵感速记、课堂/会议行动项、读书摘毛线、山谷复盘计划、云朵心情札记。前端模板弹窗会展示模板说明，便于选择。

### 4. AI 双端能力增强

端侧整理：

- 改进中文关键词提取。
- 支持中文待办识别，如“待办、行动项、需要、记得、完成、跟进、截止、提交、确认”等。
- 根据紧急、重要、规划、跟进等词判断四象限优先级。
- 断网时仍可通过后端本地规则完成整理。

云端整理：

- 请求云端模型时要求返回 `summary`、`keywords`、`todos`。
- 摘要自动压缩到适合写回笔记的长度。
- 返回非结构化内容时保留文本摘要作为降级结果。
- 云端超时、空返回、未配置 API Key 时，前端仍可使用端侧结果。

AI 结果落地：

- 摘要可写回笔记摘要字段。
- 待办可批量写入待办看板，并关联来源笔记。
- 云端返回的待办会与端侧待办合并去重。

### 5. 数据库与接口稳定性

- 为 `notes.created_at`、`notes.updated_at`、`todos.status`、`todos.deadline`、`todos.priority`、`todos.note_id` 增加索引。
- 删除笔记时同步删除关联待办，避免孤儿任务。
- 删除标签时先解除笔记关联，再删除标签。
- 保留轻量迁移逻辑，旧 SQLite 数据库启动时会自动补齐模板新增字段和索引。

## 四、测试建议

- 标题为空时保存笔记，应拦截并提示。
- 长文本与待办项混合输入后，端侧提取结果应可读。
- 断网或未配置云端 Key 时，基础搜索和端侧整理仍可用。
- 笔记插入图片后，首页卡片优先显示插入图片；未插图时显示所选背景。
- 切换 6 套主题后，首页、笔记页、待办页主色和背景应同步。
- 待办点击前方圆圈应立即完成/恢复；四个优先级圆点应只改变紧急程度。
- 在“待处理/已完成/已延期”和优先级筛选下切换任务，列表应及时刷新且不混乱。

## 五、隐私与安全说明

- 端侧整理只使用本地规则处理笔记文本，不上传云端，适合隐私敏感、弱网和低时延场景。
- 云端总结会把当前笔记标题与正文发送到配置的模型服务，用于生成摘要、关键词和行动项。
- API Key 存储在后端 `settings` 表中，课程演示可用；正式产品应加密存储并避免在日志中输出。
- 用户应避免将身份证号、密码、私密联系方式等敏感信息上传云端模型。

## 六、后续可扩展方向

- 将单标签升级为多标签关系表。
- 将图片路径从逗号字符串升级为独立图片表。
- 增加模板管理页面：新增、编辑、复制系统模板。
- 增加本地草稿缓存，后端未启动时仍能编辑。
- 为 AI 结果增加“可信度/来源”提示，便于答辩展示端侧与云端分工。
